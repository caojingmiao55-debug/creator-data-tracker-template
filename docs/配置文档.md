# 创作者数据追踪系统 - 配置文档

## 切换监测账号

只需更换对应平台的 Cookie，即可切换到其他账号进行数据监测。

---

## 配置文件位置

```
项目根目录/config.json
```

---

## 抖音配置

### 获取 Cookie

1. 浏览器打开 https://creator.douyin.com/ 并登录目标账号
2. 按 `F12` 打开开发者工具
3. 切换到 `Network` 标签
4. 刷新页面
5. 点击任意请求，在 `Headers` 中找到 `Cookie`
6. 复制完整的 Cookie 值

### 更新配置

编辑 `config.json`：
```json
{
  "douyin": {
    "enabled": true,
    "cookie": "粘贴你的Cookie到这里"
  }
}
```

---

## 小红书配置

### 获取 Cookie 和 User ID

1. 浏览器打开 https://creator.xiaohongshu.com/ 并登录目标账号
2. 按 `F12` 打开开发者工具
3. 切换到 `Network` 标签
4. 刷新页面
5. 找到 `Cookie` 并复制
6. 在 Cookie 中找到 `x-user-id-creator.xiaohongshu.com=xxx`，复制 `xxx` 部分作为 User ID

### 更新配置

编辑 `config.json`：
```json
{
  "xiaohongshu": {
    "enabled": true,
    "user_id": "你的User ID",
    "cookie": "粘贴你的Cookie到这里"
  }
}
```

## 完整配置示例

```json
{
  "douyin": {
    "enabled": true,
    "cookie": "抖音Cookie..."
  },
  "xiaohongshu": {
    "enabled": true,
    "user_id": "你的小红书UserID",
    "cookie": "小红书Cookie..."
  },
  "shipinhao": {
    "enabled": false,
    "cookie": "视频号Cookie（暂不使用）"
  },
  "settings": {
    "works_limit": 50,
    "auto_push_to_github": true,
    "github_repo": "your-username/creator-data-tracker"
  }
}
```

---

## 其他设置

| 配置项 | 说明 |
|--------|------|
| `works_limit` | 每个平台最多采集的作品数量 |
| `auto_push_to_github` | 采集后是否自动推送到 GitHub |
| `github_repo` | GitHub 仓库地址 |

---

## 更新配置后

1. 保存 `config.json`
2. 手动运行一次采集验证配置是否正确：
   ```bash
   python collect_all.py
   ```
3. 如果成功，之后会按定时任务自动采集
